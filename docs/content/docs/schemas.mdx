# Schemas

Schemas in doba aren't just versioned. They represent **different views** of the same data. A database record, a frontend payload, and an AI context window are all schemas.

## Defining schemas

Pass a map of named schemas to `createRegistry`. Each key becomes a schema identifier used in transforms and migrations.

<Tabs items={["Zod", "Valibot", "ArkType"]} groupId="schema-lib">
  <Tab value="Zod">
    ```ts
    import { createRegistry } from 'dobajs'
    import { z } from 'zod'

    const registry = createRegistry({
      schemas: {
        database: z.object({
          id: z.string(),
          email: z.string(),
          passwordHash: z.string(),
          role: z.enum(['admin', 'user']),
        }),
        frontend: z.object({
          id: z.string(),
          email: z.string(),
          role: z.enum(['admin', 'user']),
        }),
      },
      migrations: { /* ... */ },
    })
    ```

## Schema variants vs versions

doba treats all schemas equally. Whether they represent different contexts or different versions, the API is the same:

```ts
const registry = createRegistry({
  schemas: {
    // Different representations
    database: databaseSchema,
    frontend: frontendSchema,
    ai: aiSchema,
    export: exportSchema,

    // Different versions
    v1: legacySchema,
    v2: currentSchema,
  },
  migrations: {
    /* ... */
  },
})
```

## Checking schemas

```ts twoslash
import { createRegistry } from 'dobajs'
import { z } from 'zod'

const registry = createRegistry({
  schemas: {
    database: z.object({ id: z.string(), email: z.string() }),
    frontend: z.object({ id: z.string(), email: z.string() }),
  },
  migrations: {},
})
// ---cut---
const hasDb = registry.has('database')
//    ^?
```

<Callout type="info">
  Schema keys are fully typed. Passing an unknown key to `has()`, `transform()`, or `validate()` is
  a compile-time error.
</Callout>

## Identifying schemas

When you have unknown data and need to figure out which schema it belongs to, see [Schema Identification](/docs/identify). You can configure guards per schema or use a discriminator function to route unknown values.

## Standard Schema support

doba uses the [Standard Schema](https://github.com/standard-schema/standard-schema) interface, so it works with any compliant library. You can mix libraries in the same registry, but sticking with one is simpler.
